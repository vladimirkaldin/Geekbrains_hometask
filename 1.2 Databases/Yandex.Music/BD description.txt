База данных Yandex.Music

Данная база данных представляет собой имитацию структуры хранилища базы данных музыкального сервиса Yandex.Music. 
Основная задача - создание максимально структурированного каталога музыки с простым доступом к любым данным.
БД состоит из следующих таблиц:

1. users
Стандартная таблица с основными данными о пользователе. В представленном примере сервиса ситуация усложняется тем, что данные о пользователе фактически не хранятся в самом сервисе, а ведут на основной профиль Яндекса. В своей БД я сделал допущение, что профиль внутри самого сайта. Колонки в таблице стандартные - id, имя и фамилия, почта, телефон и внутренние данные по созданию и обновлению пользователя.

2. photo
База картинок, используемых пользователями, служащих иллюстрацией для исполнителей и альбомомв. 

3. profiles
Таблица данных о пользователе. Все данные стандартны. сделаны внешние ключи на таблицы users и photo для соответствующих полей. 

4. artist
Таблица исполнителей. колонки - имя, описание, картинка. внешний ключ - на таблицу картинок. Для облегчения поиска сделан индекс по имени исполнителя.

5. genres
Таблица жанров, заполнена вручную следующим запросом в соответствии с жанрами на сервисе. 

6. labels
Таблица музыкальных лейблов. Колонки - имя и картинка с внешним ключом на таблицу картинок. Заполнена вручную запросом INSERT. 

7. albums
Таблица альбомов. Колонки - исполнитель (с внешним ключом), название, год выхода, жанр (с внешним ключом), лейбл (с внешним ключом), фото (с внешним ключом). Для поиска сделан индекс по названию альбома. 

8. track
Таблица песен. Колонки - исполнитель (с внешним ключом), альбом (с внешним ключом), название и текст песни. Под тексты, в принципе, можно завести отдельную таблицу. Индекс по названию песни.

9. videos
Таблица клипов. Это не основная часть сервиса, поэтому там нет ссылок на альбом и песню, поэтому добавлен только исполнитель (с внешним ключом) и название. 

10. playlists
Плейлисты пользователей. Колонки - id пользователя (с внешним ключом), название плейлиста, id песни (с внешним ключом).

11. target_types.
Таблица типов объектов, которые можно добавить в любимые. Заполнена вручную в соответствии с сервисом.

12. likes
таблица лайков. Колонки - пользователь (с внешним ключом), id объекта, тип объекта  (с внешним ключом)
